Use these when the core pressure is **how objects and subsystems fit together**.

- [Proxy](structural/proxy.md) — pp. 236–247

## How to choose inside this family
- Need interface conversion → Adapter
- Need to separate two dimensions of variation → Bridge
- Need tree-shaped composition → Composite
- Need layered optional behavior → Decorator
- Need one simple entry point to a subsystem → Facade
- Need memory sharing for many similar objects → Flyweight
- Need controlled access to a real object → Proxy
