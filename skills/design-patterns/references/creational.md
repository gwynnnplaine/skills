# Creational Patterns

Use these when the core pressure is **how objects are created**.

- [Factory Method](creational/factory-method.md) — pp. 75–90
- [Abstract Factory](creational/abstract-factory.md) — pp. 91–105
- [Builder](creational/builder.md) — pp. 106–124
- [Prototype](creational/prototype.md) — pp. 125–138
- [Singleton](creational/singleton.md) — pp. 139–147

## How to choose inside this family
- Need subclasses to vary a created product → Factory Method
- Need matching product families → Abstract Factory
- Need staged assembly for a complex object → Builder
- Need copying of configured objects → Prototype
- Need exactly one shared instance → Singleton, used carefully
